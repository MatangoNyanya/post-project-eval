# EvalNavi（エバルナビ）デモ用モックアップ 仕様書

## 0. 本ドキュメントの目的

本仕様書は、JICA（国際協力機構）向けデモンストレーション用のモックアップアプリケーション「EvalNavi」を構築するためのものである。**実際のMLモデルやベクトルDBは使用せず、すべてダミーデータとハードコードされたロジックで動作する**が、見た目と操作感は本番同等にする。

---

## 1. プロダクト概要

### 1.1 アプリ名
**EvalNavi（エバルナビ）** — JICA事業評価ナレッジ活用システム

### 1.2 コンセプト
新規案件の基本情報（国、セクター、予算等）を入力すると、過去2000件の事後評価データに基づく機械学習モデルが「総合評価の良し悪し」を予測し、リスクが高いと判定された場合に、リスク要因ごとの過去の教訓・対策を自動提示する。

### 1.3 ターゲットユーザー
- JICA事業部門の案件担当者（案件形成段階）
- JICA評価部門の職員
- JICA経営層（デモ閲覧者）

### 1.4 技術スタック（モックアップ用）
- **フレームワーク**: Streamlit（Python）
- **言語**: Python 3.10+
- **UIコンポーネント**: Streamlit標準 + streamlit-extras（あれば）
- **チャート**: Plotly
- **データ**: Pythonの辞書・リストによるハードコード（DB不要）
- **デプロイ**: ローカル実行 or Streamlit Cloud

---

## 2. 画面構成と遷移

アプリは**単一ページ内で3ステップが順に表示される**構成とする（ページ遷移なし）。
`st.session_state`で表示ステップを管理する。

```
[STEP 1: 案件情報入力]
        │
        ▼  「スクリーニング実行」ボタン押下
[STEP 2: 予測結果 + SHAP分析]
        │
        ▼  「教訓・対策を確認する」ボタン押下
[STEP 3: リスク要因別 教訓アドバイス]
```

```python
def calculate_mock_risk(country, region, sector, subsector, scheme, budget, duration):
    """ダミーのリスク確率計算ロジック"""
    base_risk = 35  # ベースライン

    # 地域によるリスク加算
    region_risk = {
        "サブサハラアフリカ": 20,
        "南アジア": 12,
        "東南アジア": 8,
        "中南米": 5,
        "中東・北アフリカ": 10
    }
    base_risk += region_risk.get(region, 5)

    # セクターによるリスク加算
    sector_risk = {
        "運輸交通": 8,
        "水資源・防災": 6,
        "資源・エネルギー": 5,
        "都市開発・環境": 4,
        "農業・農村開発": 3,
        "ガバナンス": 7,
        "保健医療": -2,
        "教育": -3,
        "情報通信技術": 2
    }
    base_risk += sector_risk.get(sector, 0)

    # スキームによるリスク
    scheme_risk = {
        "有償資金協力": 8,
        "無償資金協力": 3,
        "技術協力": 0
    }
    base_risk += scheme_risk.get(scheme, 0)

    # 予算規模によるリスク（大きすぎても小さすぎてもリスク）
    if budget > 100:
        base_risk += 10
    elif budget > 50:
        base_risk += 5
    elif budget < 1:
        base_risk += 3

    # 事業期間によるリスク
    if duration > 72:
        base_risk += 8
    elif duration > 48:
        base_risk += 4

    # ノイズを加えて自然に見せる（±3%程度）
    import random
    random.seed(hash(country + sector + str(budget)))
    noise = random.uniform(-3, 3)
    
    return max(5, min(95, base_risk + noise))
```

## 12. デモシナリオ（推奨操作手順）

JICA向けプレゼンでのデモは、以下の手順で行うことを推奨:

### シナリオ1: デフォルト（ラオス×道路×技協）
1. アプリを開く。デフォルト値が入っている状態を確認
2. 「スクリーニングを実行する」を押す
3. 要注意確率68%、SHAP要因を説明
4. 「教訓・対策を確認する」を押す
5. リスク要因1（ラオス×制度基盤）を展開して教訓を読む
6. リスク要因2（道路×維持管理）の統計ボックスを見せる
7. 総合サマリーの3ポイントを確認

### シナリオ2: 条件変更（タンザニア×上水道×有償）
1. 「条件を変えて再分析」を押す
2. 国をタンザニア、セクターを水資源・防災、サブセクターを上水道、スキームを有償資金協力、事業費を150億円に変更
3. スクリーニング実行 → より高いリスク確率が出ることを確認
4. 教訓の内容がアフリカ×水セクターに切り替わることを確認

### シナリオ3: 低リスク案件（ベトナム×教育×技協）
1. 国をベトナム、セクターを教育、サブセクターを高等教育、スキームを技術協力、事業費を3億円、期間を36ヶ月に設定
2. スクリーニング実行 → 低いリスク確率が出ることを確認
3. 「良好な評価が見込まれます」のメッセージを見せる
4. それでも教訓の確認は推奨する旨を説明

## 13. 注意事項・制約

1. **モックアップであることの明示**: デモ時にユーザーに「これはプロトタイプであり、実際のMLモデルによる予測ではなく、ルールベースのダミーロジックで動作している」ことを明示すること
2. **教訓テキストはフィクション**: 教訓の内容は、実在の評価報告書の構造・文体を模した創作テキストである。実在の報告書からの引用ではない
3. **データの精度**: 出典として表示される報告書名・年度・ページ番号はすべてダミーである
4. **本番移行時の要件**: 実際のMLモデル、ベクトルDB、LLM APIの統合が必要。本仕様書のダミーロジック部分を置き換える形で本番化が可能な設計としている
